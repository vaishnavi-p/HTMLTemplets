Working of CI/CD

In the process of CI/CD, other applications and frameworks were also decided. But each had there own limitation and the best suited for us as Travis CI. For example, we used GIT Lab, the problem with that it was using a local server for hosting for the docker image.

To keep a clean and nice commit history we have built a different branch for updates and running and deploying the codes.
The following is the algorithm flow of Travis CI

The following is the code for deploying it and passing the condition for deploying . 
deploy:
  provider: pages
  skip_cleanup: true
  github_token: $GITHUB_TOKEN
  keep_history: true
  on:
    branch: gh-pages
    condition: $email != deploy@travis-ci.org
    
after_failure:
  - echo problem

after_deploy:
  - git push origin master
Following is the code just to run it in the branch and also just check the updated part [To reduce time in deploying]. 
git:
  depth: 2
  
branches:
  only:
   - gh-pages
